## Well Done (Keep)

1. **Parallel agent execution** — 3 agents (Backend-A, Backend-B, Frontend) ran simultaneously, reducing wall-clock time by ~60%
2. **Spec-first approach** — Spec.md already approved before implementation, reducing scope debates
3. **Test-first mindset** — 24 unit/integration tests created covering FR requirements
4. **Git safety** — No force push used, merge conflicts resolved properly
5. **Clear role separation** — Each agent owned dedicated directories, minimizing conflicts
6. **Evidence requirements met** — `evidence_ref` present in all 19+ files

---

## Improvements

1. **Merge conflict handling** — Backend-A created feature branch instead of pushing to main directly, causing merge complexity. Better: establish `main` as target from start

2. **Test coverage gap** — `constraint_evaluator.py` at 65% and `decision_logger.py` at 50% coverage. Need: more edge case tests

3. **TypeScript verification skipped** — No `package.json` / Next.js bootstrap means TS compilation not verified. Need: `npm install && tsc --noEmit`

4. **No E2E tests executed** — Only E2E test specs written (not Playwright tests run). Need: Playwright setup + actual test execution

5. **No lint/type-check in CI** — Python files pass `py_compile` but no `ruff`/`mypy`. Need: lint configuration

---

## Lessons Learned

1. **Agent Teams with shared repo needs careful branch strategy** — Empty remote caused "unrelated histories" merge issues. Solution: initialize remote repo before agents start

2. **Spec completeness matters** — Having Spec.md pre-approved saved significant back-and-forth on requirements

3. **Test coverage metrics reveal gaps** — 86% overall is good, but modules at 50-65% need attention

4. **Frontend/backend decoupling works well** — Clear API contracts let agents work independently
